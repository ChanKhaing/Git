# Layer 3 (Network Layer) & IP Addressing Study Notes

## ၁။ Layer 3 ဆိုတာ ဘာလဲ။ (What is Layer 3?)
*   **အဓိကသုံးတဲ့ စက်ပစ္စည်းများ (Core Devices):** Routers နဲ့ Layer 3 Firewalls တွေ ဖြစ်ပါတယ်။
*  
#### **ဒေတာပုံစံ (Data Unit):** ဒေတာတွေကို **Packets** အနေနဲ့ သယ်ဆောင်ပြီး အလုပ်လုပ်ပါတယ်။
*  

 **အဓိက လုပ်ဆောင်ချက်များ (Key Functions):** 
    *   ဒေတာတွေရဲ့ ခေါင်းစီး (Header) မှာ **Source IP** (ပို့သူ) နဲ့ **Destination IP** (လက်ခံသူ) လိပ်စာတွေကို ကပ်ပေးပါတယ်။
#### what he really care
 (Routing):** Network တစ်ခုကနေ တစ်ခုကို ဒေတာတွေ ရောက်သွားဖို့ အကောင်းဆုံးနဲ့ အမြန်ဆုံး လမ်းကြောင်းကို ရှာဖွေရွေးချယ်ပေးပါတယ်။
*   **ဒေတာပို့ဆောင်မှု ပုံစံ (Delivery Style):** Best-Effort Delivery (Connectionless) စနစ် ဖြစ်ပါတယ်။ လမ်းကြောင်းပေါ် အမြန်ဆုံးရောက်ဖို့ပဲ အဓိကထားပြီး လမ်းမှာ ဒေတာပျောက်ဆုံးသွားရင် ပြန်တောင်းတာမျိုး (Error Recovery) ကို ၎င်းကိုယ်တိုင် တာဝန်မယူပါဘူး

 (Layer 4 TCP ရဲ့ အလုပ်ဖြစ်ပါတယ်)။ သို့သော် IP Header အစိတ်အပိုင်း ပျက်စီးသွားရင်တော့ Checksum စနစ်နဲ့ စစ်ဆေးပြီး အဲဒီ Packet ကို ဖျက်ချ (Drop) ပစ်ပါတယ်။

*   **လက်တွေ့အသုံးချမှု (Real-world Application):** Real-time အလုပ်လုပ်ရတဲ့ Voice/Video Calls (VC) တွေမှာ အသုံးများပါတယ်။ လိုင်းနည်းနည်းထစ်ရင် ထစ်ပါစေ ဒေတာအမြန်ဆုံးရောက်ဖို့ လိုအပ်တဲ့ နေရာတွေမှာ အဓိက အကျိုးပြုပါတယ်။
*   **စနစ်လမ်းကြောင်း (Directionality):** Bidirectional (Two-way) စနစ် ဖြစ်ပါတယ်။ Source နဲ့ Destination အကြား အပြန်အလှန် ဒေတာစီးဆင်းမှု ရှိပါတယ်။
####  ** IP သတ်မှတ်ခြင်း (Configuration):** IP address တွေကို ကိုယ်တိုင် လက်နဲ့ရိုက်ထည့်တာ (**Static**) သော်လည်းကောင်း၊ DHCP Server ကနေ အလိုအလျောက် ချပေးတာ (**Dynamic**) သော်လည်းကောင်း သတ်မှတ်နိုင်ပါတယ်။

## ၂။ IP Addressing ဗိသုကာ (IP Addressing Architecture)

### က။ Classful Addressing (ပုံသေစနစ်)
IP Address တွေကို သတ်မှတ်ထားတဲ့ အုပ်စု (Classes) တွေအတိုင်း ခွဲခြားပြီး Default Subnet Mask တွေကို ပုံသေ သတ်မှတ်သုံးစွဲတဲ့ စနစ်ဟောင်း ဖြစ်ပါတယ်။
*   **Class A:** `1.0.0.0` မှ `126.255.255.255` အထိ (Default: `/8` သို့မဟုတ် `255.0.0.0`)
*   **Class B:** `128.0.0.0` မှ `191.255.255.255` အထိ (Default: `/16` or `255.255.0.0`)
*   **Class C:** `192.0.0.0` မှ `223.255.255.255` အထိ (Default: `/24` or `255.255.255.0`)
*   *အားနည်းချက်:* Block အရွယ်အစားတွေက ပုံသေဖြစ်နေတဲ့အတွက် လိုအပ်တာထက် IP address တွေ အမြောက်အမြား အလဟဿ ဖြုန်းတီးပစ်သလို ဖြစ်စေပါတယ်။

### ခ။ Classless Addressing / CIDR (လွတ်လပ်သောစနစ်)
Class တွေရဲ့ ပုံသေစည်းကမ်းကို မလိုက်နာဘဲ မိမိလိုအပ်တဲ့ Host အရေအတွက်အလိုက် Subnet Mask ကို စိတ်ကြိုက်ပြောင်းလဲပြီး Subnetting ဆောက်နိုင်တဲ့ စနစ်ဖြစ်ပါတယ်။ IP ဖြုန်းတီးမှုကို အကောင်းဆုံး တားဆီးပေးနိုင်ပြီး Network တစ်ခုနဲ့တစ်ခုကြား လုံခြုံရေးအရ စည်းခြား (Isolate) ရန်လည်း အသုံးပြုပါတယ်။

### ဂ။ Private IP Address Ranges (LAN အတွင်းသုံးအုပ်စု)
Local Network (LAN) ထဲမှာပဲ သုံးခွင့်ရှိပြီး Internet ပေါ် တိုက်ရိုက်ထွက်လို့ မရတဲ့ သီးသန့်ဖယ်ထားသော IP Range များ ဖြစ်ကြပါတယ်-
*   **Class A:** `10.0.0.0` မှ `10.255.255.255` အထိ (`10.` နဲ့ စတာအားလုံး)
*   **Class B:** `172.16.0.0` မှ `172.31.255.255` အထိ (`172.16` မှ `172.31` အထိ)
*   **Class C:** `192.168.0.0` မှ `192.168.255.255` အထိ (`192.168.` နဲ့ စတာအားလုံး)

## ၃။ Layer 3 ရဲ့ အဆင့်မြင့် အလုပ်လုပ်ပုံများ (Advanced Mechanics)

### Subnet Mask နှင့် Wildcard Mask နှိုင်းယှဉ်ချက်
*   **Subnet Mask:** IP address တစ်ခုရဲ့ Network အပိုင်း (`1` bits) နဲ့ Host အပိုင်း (`0` bits) ကို ခွဲခြားသတ်မှတ်ပေးပါတယ်။
*   **Wildcard Mask:** Subnet Mask ရဲ့ အပေါင်းလက္ခဏာပြောင်းပြန် (Bitwise NOT) ဖြစ်ပါတယ်။ Subnet Mask မှာ `1` ရှိရင် Wildcard Mask မှာ `0` ဖြစ်ပြီး၊ ပြောင်းပြန်လှန်ပစ်တဲ့ စနစ်ပါ။ OSPF Routing နဲ့ Access Control Lists (ACL) firewall rules တွေမှာ traffic စစ်ဆေးဖို့ သုံးပါတယ်။
    *   *ဥပမာ (Binary အမြင်):* 
        *   Subnet Mask: `11111111.11000000.00000000.00000000` (`255.192.0.0`)
        *   Wildcard Mask: `00000000.00111111.11111111.11111111` (`0.63.255.255`)

### TTL (Time to Live) - Packet ၏ သက်တမ်း
Network ထဲမှာ Router တွေရဲ့ လမ်းကြောင်းမှားပြီး Packet တွေ ပတ်ချာလည် (Routing Loop) မဖြစ်အောင် တားဆီးပေးတဲ့ စနစ်ပါ။ Packet တစ်ခုက Router တစ်လုံးကို ဖြတ်ကျော် (ခုတ်မောင်း) တိုင်း တန်ဖိုး ၁ လျော့ကျသွားပါတယ်။ တကယ်လို့ `TTL = 0` ဖြစ်သွားရင် Router က Packet ကို ဖျက်ဆီး (Drop) ပစ်လိုက်ပါတယ်။

### ARP (Address Resolution Protocol)
မိမိသိထားတဲ့ Layer 3 IP Address ကို သုံးပြီး Layer 2 MAC Address ရှာဖွေပြောင်းလဲပေးတဲ့ "တံတား" ပရိုတိုကော ဖြစ်ပါတယ်။ စက်တစ်လုံးက ဒေတာပို့ချင်ပေမယ့် သူ့ရဲ့ ARP Table ထဲမှာ လက်ခံမယ့်စက်ရဲ့ MAC Address မရှိသေးတဲ့အခါ-
1.  **ARP Request:** Broadcast စနစ်နဲ့ တစ်ကွန်ရက်လုံးကို အော်မေးပါတယ်။ ("ဒီ IP ရှိတဲ့သူ ဘယ်သူလဲ?")
2.  **ARP Reply:** သက်ဆိုင်တဲ့ Target စက်ကနေပဲ Unicast (တိုက်ရိုက်) စနစ်နဲ့ သူ့ရဲ့ MAC Address ကို ပြန်လည်ဖြေကြားပါတယ်။

### Ping
အဲဒီ IP address ဆီကို ဒေတာစီးဆင်းမှု တကယ်ရှိမရှိ၊ ကွန်ရက်ချိတ်ဆက်မှု ရမရ လက်တွေ့ စမ်းသပ်စစ်ဆေးတဲ့ utility tool ဖြစ်ပါတယ်။
