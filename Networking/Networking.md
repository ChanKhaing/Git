## 1. Binary to Decimal (The 8-Bit Grid)

IPv4 Address တွေနဲ့ Subnetting တွက်ဖို့အတွက် ညာဘက်ကနေစပြီး $2$ ရဲ့ ထပ်ညွှန်း (Power) တွေအတိုင်း $2$ ဆစီ တိုးသွားတဲ့ အံဆွဲ ၈ ခု ရှိပါတယ်။

| 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |

### 💡 ကျွန်တော် မှားခဲ့တဲ့အချက် (My First Mistake)
* **ထင်မိတာ:** `1000` ကို မြင်တော့ ငယ်ငယ်က အလေ့အထအတိုင်း "တစ်ထောင်" လို့ပဲ တွေးမိပြီး Binary base အနေနဲ့ တွက်ဖို့ ခဏ တွေဝေသွားခဲ့တယ်။
* **အမှန်:** နေရာ ၄ လုံးမြောက်က $2^3$ ဖြစ်တဲ့အတွက် ကွန်ပျူတာမှာ `1000` ဆိုတာ **`8`** ဖြစ်ပါတယ်။

### 🛠️ စမ်းသပ်ပုစ္ဆာ (Real Example)
* **ဂဏန်း:** `195` ကို Binary ပြောင်းချင်ရင်:
* $128 + 64 + 2 + 1 = 195$ မို့လို့ $1$ ရှိတဲ့ အံဆွဲတွေပဲ ဖွင့်မယ်။
* **ရလဒ်:** `11000011`

## 2. Hexadecimal To Decimal (အမှားကနေ သင်ခန်းစာရပုံ)

Hexadecimal ($0-9$ အပြင် $A$ မှ $F$) ကို Decimal ပြန်ပြောင်းတဲ့အခါ **"၄ လုံးဖြတ်နည်း"** ကို သုံးပါတယ်။

### 💡 ကျွန်တော် မှားခဲ့တဲ့အချက် (The "C5" Trap)
ကျွန်တော် `C5` ကို Decimal ပြောင်းတုန်းက **`213`** ဆိုပြီး အဖြေထွက်ခဲ့တယ်။ ဘာလို့မှားသွားတာလဲ?

1. **မှားယွင်းတဲ့ Hex တန်ဖိုး:** `C` ကို $13$ လို့ မှားယူမိခဲ့တယ် (အမှန်က $A=10, B=11, C=12$ ဖြစ်ရမယ်)။
2. **မှားယွင်းတဲ့ Grid အစဥ်လိုက်:** အောက်မှာ အံဆွဲတန်ဖိုး ချရေးတဲ့အခါ **$32$** ကို ကျော်ပြီး `128 64 16...` ဆိုပြီး စီမိသွားခဲ့တယ်။

### 🛠️ မှန်ကန်တဲ့ တွက်ချက်ပုံစနစ် (How to fix it)
`C5` ကို အမှန်အတိုင်း ပြန်ခွဲရင်:
1. **`C` (12) ကို ၄ လုံးဖြတ်မယ်:** $8 + 4 = 12$ ဖြစ်လို့ $\rightarrow$ **`1100`**
2. **`5` ကို ၄ လုံးဖြတ်မယ်:** $4 + 1 = 5$ ဖြစ်လို့ $\rightarrow$ **`0101`**

အဲဒီ ၄ လုံးစီကို ပေါင်းလိုက်တော့ ၈ လုံးတွဲ Binary က **`11000101`** ရတယ်။ ဒါကိုမှ အံဆွဲအမှန်အောက်မှာ သွားစီတယ်-

| 128 | 64 | 32 | 16 | 8 | 4 | 2 | 1 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| **1** | **1** | 0 | 0 | 0 | **1** | 0 | **1** |

* **အဖြေမှန်:** $128 + 64 + 4 + 1 =$ **`197`**

## 3. အောင်မြင်သွားတဲ့ နောက်တစ်ဆင့် (`D1` Challenge)

အပေါ်က အမှား ၂ ခုလုံးကို ပြင်ပြီးတဲ့နောက်မှာ **`D1`** ကို ကိုယ်တိုင် ကွက်တိ တွက်နိုင်ခဲ့ပါတယ်။

1. **`D` ရဲ့ တန်ဖိုးက $13$:** Binary ပြောင်းရင် $\rightarrow$ **`1101`**
2. **`1` ရဲ့ တန်ဖိုး:** Binary ပြောင်းရင် $\rightarrow$ **`0001`**
3. **ပေါင်းလိုက်ရင်:** `11010001`
4. **Grid နဲ့ ပေါင်းထုတ်ခြင်း:** $128 + 64 + 16 + 1 =$ **`209`** (လုံးဝ ကွက်တိ!)

## 4. OSI 7 Layers & Data Encapsulation Process 🔄

Network ပေါ်မှာ Data တွေ ဘယ်လို သွားလာလဲဆိုတာကို Layer အလိုက် ခွဲခြားနားလည်ဖို့ ထပ်မံလေ့လာခဲ့ပါတယ်။ 



[Image of data encapsulation OSI model]


### 📦 Layer အလိုက် Data ပုံစံပြောင်းလဲသွားပုံ (Encapsulation Flow)

1. **Layers 7, 6, 5 (Application, Presentation, Session):** User က စတင်လိုက်တဲ့ အဆင့်ဖြစ်ပြီး သာမန် `Data` အဖြစ် ရှိနေမယ်။
2. **Layer 4 - Transport Layer (`Segment`):** * ဒီအဆင့်ရောက်ရင် Data အကြီးကြီးတွေကို တစ်ထုပ်စာ (Standard 1500 Bytes - MTU အရ) စီ ခွဲပစ်လိုက်တယ်။
   * ပြီးရင် **Source Port** နဲ့ **Destination Port** (TCP/UDP header) ကပ်ပေးလိုက်တဲ့အခါ ဒါကို **`Segment`** လို့ ခေါ်ပါတယ်။
3. **Layer 3 - Network Layer (`Packet`):** * အပေါ်က ဆင်းလာတဲ့ Segment ကို **Source IP** နဲ့ **Destination IP** ထပ်မံ အုပ်လိုက်တယ်။ ဒါကို **`Packet`** လို့ ခေါ်ပါတယ်။
4. **Layer 2 - Data Link Layer (`Frame`):** * အဲဒီ Packet ကိုမှ Hardware ကတ်ပြားတွေ သိနိုင်ဖို့ **Source MAC** နဲ့ **Destination MAC** Address တွေ ထပ်ကပ်ပေးလိုက်တယ်။ ဒါကို **`Frame`** လို့ ခေါ်ပါတယ်။
5. **Layer 1 - Physical Layer (`Bits`):** * နောက်ဆုံးမှာတော့ အဲဒီ Frame ကို လျှပ်စစ်စီးကြောင်း (0 နဲ့ 1) `Bits` တွေအဖြစ် ပြောင်းပြီး ကြိုးတွေ သို့မဟုတ် လှိုင်းတွေကနေတစ်ဆင့် Hardware ဆီ လှမ်းပို့လိုက်ပါတယ်။
