ခေါင်းစဉ် - Network Switch တစ်လုံး၏ အလုပ်လုပ်ပုံနှင့် Layer 2 အခြေခံများ
### ၁။ Switch တစ်လုံး၏ အဓိက အလုပ် ၃ ခု
Address Learning (လိပ်စာမှတ်သားခြင်း): ဝင်လာသမျှ Data (Frame) တွေရဲ့ ပို့သူ MAC Address ကို ကြည့်ပြီး ဘယ် Port မှာ ဘယ်ကွန်ပျူတာ ရှိလဲဆိုတာ Table ထဲ မှတ်သားခြင်း။

Forward/Filter Decision (လမ်းကြောင်းရွေးချယ်ခြင်း): လက်ခံမည့်သူ (Destination MAC) ကို ကြည့်ပြီး သက်ဆိုင်ရာ Port တစ်ခုတည်းဆီပဲ တိုက်ရိုက်ပို့ပေးခြင်း (Forward)၊ မသက်ဆိုင်တဲ့ Port တွေဆီ Data မရောက်အောင် ပိတ်ပင်ခြင်း (Filter)။

Loop Avoidance (ပတ်ချာလည်မဖြစ်အောင် တားဆီးခြင်း): Network ထဲမှာ Data တွေ ပတ်ချာလည်ပတ်ပြီး Network ကျမသွားအောင် STP (Spanning Tree Protocol) နဲ့ ထိန်းကျောင်းခြင်း။

###### ၂။ ကွန်ပျူတာ A နှင့် B အကြား switch အသစ်မာ ပထမဆုံးအကြိမ် Data ပို့ဆောင်ခြင်းအဆင့်ဆင့်
အစပိုင်းမှာ Switch ရဲ့ Table ထဲမှာ ဘာမှမရှိသေးပါ။ ကွန်ပျူတာတွေမှာလည်း IP သာသိပြီး တစ်ဖက်လူရဲ့ MAC Address ကို မသိသေးပါ။

အဆင့် (၁) - ARP Request ပို့ခြင်း: ကွန်ပျူတာ A ကနေ B ရဲ့ MAC Address ကို သိချင်လို့ Network တစ်ခုလုံးကို Broadcast (FF:FF:FF) စနစ်နဲ့ အော်မေးပါတယ်။

အဆင့် (၂) - Switch က စတင်မှတ်သားခြင်း: အဲဒီ Data ဟာ Port 1 ကနေ ဝင်လာတဲ့အတွက် Switch က "Port 1 မှာ ကွန်ပျူတာ A ရှိတယ်" ဆိုပြီး Table ထဲမှာ စမှတ်ပါတယ်။

အဆင့် (၃) - Broadcast Flooding: ဒါဟာ လူတိုင်းဆီပို့တဲ့ Broadcast ဖြစ်လို့ Switch က Port 1 ကလွဲလို့ ကျန်တဲ့ Port အားလုံးဆီ ဖြန့်ပို့လိုက်ပါတယ်။

အဆင့် (၄) - ARP Reply ပြန်ခြင်း: ကွန်ပျူတာ B က သူ့ဆီပို့မှန်းသိတော့ A ထံသို့ တိုက်ရိုက် Unicast စနစ်နဲ့ ပြန်ဖြေပါတယ်။

အဆင့် (၅) - Table ပြည့်စုံသွားခြင်း: B ရဲ့ အဖြေဟာ Port 2 ကနေ ဝင်လာတဲ့အတွက် Switch က "Port 2 မှာ ကွန်ပျူတာ B ရှိတယ်" လို့ ထပ်မှတ်လိုက်ပါတယ်။ အခုဆိုရင် A ရော B ရောကို Switch က သိသွားပါပြီ။

##### ၃။ Switching Modes ၃ မျိုး (Data ပို့ဆောင်ပုံစံများ)
Cut-Through: ဝင်လာတဲ့ Frame ရဲ့ Destination MAC (လိပ်စာ) ကို မြင်တာနဲ့ ကျန်တဲ့ Data တွေအကုန် ဝင်လာတာကို မစောင့်ဘဲ ရှေ့ကို ချက်ချင်း လွှဲပို့ပေးတာပါ။ (အမြန်ဆုံးဖြစ်ပေမဲ့ Error စစ်လို့မရပါ)

Store-and-Forward: Frame တစ်ခုလုံး အပြည့်အဝ ဝင်လာတဲ့အထိ စောင့်၊ ပြီးမှ Error ပါမပါ စစ်ဆေး (CRC checking လုပ်) ပြီးမှ ရှေ့ကို ပို့ပေးတာပါ။ (စိတ်အချရဆုံးဖြစ်ပေမဲ့ အနည်းငယ်နှေးတယ်)

Fragment-Free: အထက်ပါ ၂ ခုရဲ့ ကြားထဲကပါ။ Frame ရဲ့ ပထမဆုံး 64 Bytes ကိုပဲ စစ်ဆေးပြီး Error မရှိရင် ရှေ့ကို ပို့ပေးတာ ဖြစ်ပါတယ်။

၄။ Transmission Modes (ဆက်သွယ်ရေးပုံစံ ၃ မျိုး)
Simplex: တစ်ဖက်သတ်ပဲ ပို့လို့ရတဲ့စနစ် (ဥပမာ- ကွန်ပျူတာကနေ Monitor ဆီ Signal ပို့သလိုမျိုး၊ ပြန်ပြောလို့မရ)။

Half-Duplex: နှစ်ဖက်လုံး ပြောလို့ရပေမဲ့ တစ်ချိန်မှာ တစ်ယောက်ပဲ ပြောလို့ရတဲ့စနစ် (ဥပမာ- Walkie-Talkie လိုမျိုး၊ တစ်ဖက်ကပြောရင် တစ်ဖက်က နားထောင်ရမယ်၊ ပြိုင်ပြောရင် Data တိုက်မိမယ်)။

Full-Duplex: နှစ်ဖက်လုံး တစ်ပြိုင်နက်တည်း ပြောလို့၊ ပို့လို့ရတဲ့စနစ် (ဥပမာ- ဖုန်းပြောသလိုမျိုး၊ ခေတ်ပေါ် Switch တွေက ဒီစနစ်ကို သုံးပါတယ်)။

##### ၅။ Domain နှစ်မျိုးအကြောင်း
Collision Domain: Data တွေ တိုက်မိနိုင်တဲ့ နယ်ပယ်။ Switch တစ်လုံးရဲ့ Port တိုင်းဟာ သီးခြား Collision Domain တစ်ခုစီ ဖြစ်ပါတယ်။ ဒါကြောင့် Switch မှာ Data ချင်း တိုက်မိခဲပါတယ်။ (Hub မှာတော့ Domain တစ်ခုတည်းမို့ ခဏခဏ တိုက်မိတတ်ပြီး CSMA/CD စနစ်နဲ့ ထိန်းရပါတယ်)။

Broadcast Domain: Broadcast အော်လိုက်ရင် ရောက်သွားနိုင်တဲ့ နယ်ပယ်။ Switch တစ်လုံးလုံး (VLAN မခွဲမချင်း) ဟာ Broadcast Domain တစ်ခုတည်း ဖြစ်ပါတယ်။ (ဒါကို ပိုင်းခြားချင်ရင် Router သုံးရပါတယ်)။

###### ၆။ Data Link Layer (Layer 2) ၏ အလုပ် ၅ ခု
Framing: Packet တွေကို Frame အဖြစ် ပုံစံပြောင်းခြင်း။

Physical Addressing: Hardware လိပ်စာ (MAC Address) တပ်ပေးခြင်း။

Flow Control: ပို့တဲ့နှုန်းနဲ့ လက်ခံတဲ့နှုန်း မျှတအောင် ထိန်းပေးခြင်း။

Error Control: Data ပျက်စီးသွားရင် ရှာဖွေဖော်ထုတ်ခြင်း။

Access Control: ကြိုးလိုင်းတစ်ခုတည်းကို ကွန်ပျူတာအများကြီး သုံးနေရင် ဘယ်သူ အရင်ပို့မလဲဆိုတာ ထိန်းချုပ်ပေးခြင်း။
