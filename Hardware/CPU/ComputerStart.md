 ###### how computer really work i understand 
ကွန်ပျူတာ အလုပ်လုပ်ပုံကို အောက်ခြေ Hardware ကနေ အပေါ်ဆုံး Software အထိ စနစ်တကျ အလွှာလိုက် (Layers) မြင်ကြည့်ရင် ပိုပြီး ပြည့်စုံသွားပါလိမ့်မယ်။ သင်သိထားတဲ့အထဲက ကွင်းဆက်ပြတ်နေတဲ့ နေရာလေးတွေကို ဖြည့်စွက်ပြီး အစဉ်လိုက် ရှင်းပြပေးပါ့မယ်။ကွန်ပျူတာတစ်လုံး ခလုတ်နှိပ်လိုက်တာနဲ့ အလုပ်လုပ်သွားတဲ့ ပုံစံက ဒီအတိုင်း ဖြစ်ပါတယ် 
-၁။ Hardware အလွှာ (CPU, RAM, Motherboard)ဒါက အောက်ခြေအကျဆုံး ရုပ်ဝတ္ထုပစ္စည်းတွေ ဖြစ်ပါတယ်။CPU: ကွန်ပျူတာရဲ့ ဦးနှောက်။ သူက တွက်ချက်မှု မှန်သမျှကို လုပ်တယ်။RAM: ကွန်ပျူတာရဲ့ ယာယီမှတ်ဉာဏ်။ CPU တွက်ချက်မယ့် Data တွေကို ခဏသိမ်းပေးထားတယ်။Motherboard: ဒီပစ္စည်းအားလုံးကို လာစိုက်ရတဲ့ အောက်ခံပတ်လမ်းပြားကြီး။

၂။ BIOS / UEFI အလွှာ စက်ခလုတ်နှိပ်လိုက်တာနဲ့ CPU က အရင်ဆုံး သွားနှိုးတာက Motherboard ပေါ်က BIOS ချစ်ပ်ပြားကို ဖြစ်ပါတယ်။BIOS က Hardware တွေ (RAM ကောင်းလား၊ SSD ရှိလား) အကုန်လိုက်စစ်ဆေးတယ်။အကုန်အိုကေပြီဆိုရင် Storage (Hard Drive / SSD) ရဲ့ ထိပ်ဆုံးမှာရှိတဲ့ Bootloader (ဥပမာ - Linux မှာဆိုရင် GRUB ၊ Windows မှာဆိုရင် Windows Boot Manager) ဆီကို အာဏာလွှဲပေးလိုက်ပါတယ်။ဖြည့်စွက်ချက်: BIOS နဲ့ Kernel ကြားထဲမှာ "Bootloader" ဆိုတဲ့ ပေါက်ကူးတံတားလေး တစ်ခု ရှိပါသေးတယ်။ ဒီ Bootloader ကမှတစ်ဆင့် Hard Drive ထဲက Kernel ကို ရှာပြီး RAM ပေါ်ကို ဆွဲတင်ပေးတာ ဖြစ်ပါတယ်။
၃။ Kernel အလွှာ (သင်ပြောတဲ့ တတိယအဆင့်)Kernel ဆိုတာ Operating System (OS) တစ်ခုလုံးရဲ့ နှလုံးသား ဖြစ်ပါတယ်။ (Linux Kernel, Windows NT Kernel စသဖြင့်)Bootloader က Kernel ကို RAM ပေါ်တင်ပေးလိုက်တာနဲ့ Kernel က စက်ရဲ့ အာဏာအားလုံးကို သိမ်းပစ်လိုက်ပါတယ်။သူ့ရဲ့ အဓိကအလုပ်က Software တွေနဲ့ Hardware တွေကြားမှာ စကားပြန်လုပ်ပေးရတာ ဖြစ်ပါတယ်။ ဥပမာ - ပရိုဂရမ်တစ်ခုက Files သွားသိမ်းချင်ရင် Kernel ကို လှမ်းပြောရတယ်။ Kernel ကမှတစ်ဆင့် Hard Drive ကို သွားခိုင်းတာ ဖြစ်ပါတယ်။
၄။ သင်သိထားတဲ့အထဲကို ထပ်ဖြည့်ရမည့် အပေါ်ဆုံးအလွှာ (OS Userspace & Applications)Kernel ပွင့်လာပြီးနောက်ပိုင်းမှသာ ကျွန်တော်တို့ မျက်စိနဲ့မြင်ရတဲ့ အပိုင်းတွေ တက်လာတာ ဖြစ်ပါတယ်။Operating System Services (OS): Windows Graphical User Interface (Desktop ကြီး) သို့မဟုတ် Linux ရဲ့ Shell Command Prompt တက်လာပါတယ်။Applications: အဲဒီနောက်ပိုင်းမှ ကျွန်တော်တို့ သုံးချင်တဲ့ Google Chrome, Discord, VMware စတဲ့ Software တွေကို ဖွင့်ပြီး သုံးလို့ရတာ ဖြစ်ပါတယ်။
အကျဉ်းချုပ် ကွင်းဆက် (Summary Flow)သင်အလွယ်မှတ်ရအောင် ပုံသေနည်း ပြန်ထုတ်ပေးရရင် -{Hardware (CPU/RAM)} {BIOS/UEFI} \{Bootloader} \{Kernel} \{Operating System} \{Applications (Software)}
သင်က Linux သမားမို့လို့ ပိုနားလည်အောင် ပြောရရင် -Linux စက်တစ်လုံး ပွင့်လာတဲ့အခါ BIOS က စက်နှိုးတယ် $ GRUB (Bootloader) က တက်လာတယ်  GRUB က Vmlinuz (Kernel) ကို RAM ပေါ်တင်ပေးတယ်  Kernel က စနစ်တစ်ခုလုံးရဲ့ ပထမဆုံးပရိုဂရမ်ဖြစ်တဲ့ /sbin/init (Systemd) ကို နှိုးလိုက်ပြီးမှ Desktop ကြီး ပွင့်လာတာ ဖြစ်ပါတယ်။

၁။ "CPU က အလုပ်လုပ်ပြီး BIOS က မတက်လာရင် မီးပဲလင်းတယ်" 
-->(မှန်ကန်ပါတယ်)ဘာဖြစ်လို့လဲ: ပါဝါခလုတ်နှိပ်လို့ မီးလင်းတယ် (မီးရတယ်)၊ ဒါပေမယ့် Screen မှာ ဘာစာသားမှ မပေါ်ဘဲ မည်းမှောင်နေရင် ဒါဟာ BIOS က Hardware စစ်တဲ့အဆင့် (POST) မှာ တင် တန့်နေလို့ ဖြစ်ပါတယ်။ အများအားဖြင့် RAM ဂျီးတင်နေတာ၊ Motherboard ချို့ယွင်းတာ သို့မဟုတ် BIOS Firmware ကိုယ်တိုင် ပျက်စီးနေရင် ဒီလို ဖြစ်တတ်ပါတယ်။
၂။ "BIOS က ရပြီး GRUB က အလုပ်မလုပ်ရင် SSD က Kernel တွေကို မရှာနိုင်ဘူး" 
--> (မှန်ကန်ပါတယ်)ဘာဖြစ်လို့လဲ: BIOS ပွင့်လာပြီး Screen မှာ စာသားတွေ ပေါ်လာတယ်၊ ဒါပေမယ့် Error loading operating system သို့မဟုတ် No bootable device found လို့ ပြနေရင် BIOS က သူ့အလုပ်သူ လုပ်ပြီးပြီ။ ဒါပေမယ့် SSD ထဲက တံခါးစောင့် (GRUB သို့မဟုတ် Windows Boot Manager) ကို ရှာမတွေ့တာ သို့မဟုတ် အဲဒီ Bootloader ပျက်စီးနေလို့ Kernel ရှိတဲ့နေရာကို မညွှန်ပြနိုင်တာ ဖြစ်ပါတယ်။
၃။ "GRUB က လုံလောက်ပြီး RAM က မလုံလောက်ရင် SSD က Kernel ကို ရှာတွေ့ရင်တောင် Loading ကြာနေမယ်၊ Logo ပြပြီး..." 
--> (၉၀% မှန်ပါတယ်၊ နည်းနည်းလေး ဖြည့်စွက်ပါရစေ)ဖြည့်စွက်ချက်: Kernel ဖိုင်ဆိုတာ တကယ်တော့ ဖိုင်ဆိုက် (File Size) အရမ်းသေးပါတယ်။ (ဥပမာ - Linux Kernel တစ်ခုလုံးဟာ 5MB ကနေ 10MB ဝန်းကျင်ပဲ ရှိလို့ RAM ဘယ်လောက်နည်းနည်း RAM ပေါ်ကို ခဏလေးနဲ့ ရောက်သွားပါတယ်)။တကယ် ဖြစ်တတ်တာက: စက်ဖွင့်ရင် Logo ကြီး ပြပြီး ပိတ်သွားတာ (သို့မဟုတ်) Blue Screen / Black Screen တက်ပြီး အကြာကြီး လည်နေတာဟာ—RAM မလောက်လို့ မဟုတ်ဘဲ RAM အကြမ်းထည် ပျက်စီးနေလို့ (Hardware Error) ဖြစ်နိုင်ပါတယ်။ဒါမှမဟုတ် Kernel က RAM ပေါ်ရောက်ပြီး စက်ရဲ့ Hardware တွေ (ဥပမာ - Graphic Card သို့မဟုတ် Motherboard Drivers) ကို လိုက်နှိုးတဲ့အခါ Driver ဖိုင် ချို့ယွင်းနေလို့ ရှေ့ဆက်မတက်ဘဲ Logo နားမှာတင် ရပ်နေတာ ဖြစ်တတ်ပါတယ်။
###### when you got error 
ကွန်ပျူတာတစ်လုံးက grub> ဆိုတဲ့ စာတန်းကြီးပြပြီး ပိတ်နေပြီဆိုရင် အထဲက Linux OS ထဲကို ပုံမှန်အတိုင်း ဝင်လို့မရတော့ပါဘူး။ အဲဒီအခါ IT Support သမားတစ်ယောက်အနေနဲ့ အောက်ပါအတိုင်း အဆင့်ဆင့် ပြင်ရပါတယ်။

၁။ Live USB နဲ့ စက်နှိုးခြင်း (Boot from Live USB)
ပထမဆုံး Ubuntu သို့မဟုတ် တခြား Linux တစ်ခုခု ထည့်ထားတဲ့ Live USB (စမ်းသပ်သုံးစွဲလို့ရတဲ့ USB) ကို စက်မှာ စိုက်ပြီး ဖွင့်လိုက်ပါတယ်။ ဒါဆိုရင် ပျက်နေတဲ့ စက်ကြီးထဲမှာ တင် USB ထဲက Linux စနစ်ယာယီတစ်ခု ပွင့်လာပါလိမ့်မယ်။

၂။ chroot (Change Root) ဆိုတာ ဘာလဲ။
အခုအချိန်မှာ သင် ရောက်နေတာက USB ထဲက Linux ထဲမှာ ဖြစ်ပါတယ်။ ဒါပေမယ့် သင် ပြင်ချင်တာက စက်ထဲက ပျက်နေတဲ့ SSD ဖြစ်နေပါတယ်။

ဒီနေရာမှာ chroot (Change Root) ဆိုတဲ့ Command ကို သုံးရပါတယ်။

ဒီ Command ရဲ့ သဘောတရားက "ငါ အခု သုံးနေတဲ့ နေရာကို ခေတ္တခဏ မေ့ထားပြီး၊ စက်ထဲက ပျက်နေတဲ့ SSD ရဲ့ / (Root Partition) ထဲကို Control ရွှေ့ပြီး ဝင်လိုက်မယ်" လို့ ခိုင်းလိုက်တာ ဖြစ်ပါတယ်။

ဒါဆိုရင် သင်က USB ထဲကနေ စက်ထဲက SSD ထဲကို လှမ်းပြီး စုန်းဝင်ပူးလိုက်သလိုမျိုး စက်ထဲက ပျက်နေတဲ့ Linux စနစ်ကို စတင် ခြယ်လှယ်လို့ ရသွားပါပြီ။

၃။ update-grub နဲ့ ပြန်ပြင်ခြင်း
SSD ထဲကို chroot နဲ့ ရောက်သွားပြီဆိုရင်တော့ ပျက်နေတဲ့ GRUB ကို အသစ်ပြန်ဖြစ်အောင် Command ရိုက်ပြီး ခိုင်းရပါတော့တယ်။

 #### စည်းကမ်းချက် (Golden Rule)
ဒီစမ်းသပ်မှုကို သင့်ရဲ့ အပြင်က စက်အစစ် (Physical Host OS) ပေါ်မှာ လုံးဝ မလုပ်ပါနဲ့။ မိမိစက်ထဲက VMware သို့မဟုတ် VirtualBox ထဲမှာ တင်ထားတဲ့ Linux (ဥပမာ Ubuntu သို့မဟုတ် Debian) စက်အယောင်ဆောင် (Virtual Machine) ထဲမှာပဲ လုပ်ပါမယ်။ ဒါဆိုရင် ဘယ်လောက်ပဲ ပျက်ပျက် မိနစ်ပိုင်းအတွင်း Snapshot နဲ့ ပြန်ခေါ်လို့ ရပါတယ်ဗျာ။🛠️ စမ်းသပ်ခန်း အပိုင်း (၁) - GRUB ကို ဖျက်ဆီးပြီး ပြန်ပြင်ခြင်းတံခါးစောင့် GRUB ကို တမင်တကာ သတ်ပစ်လိုက်ပြီး စက်ပွင့်မလာအောင် လုပ်ပြီးမှ Live USB နဲ့ ပြန်ကယ်တင်ကြပါမယ်။[အဆင့် ၁] တမင်တကာ ဖျက်ဆီးခြင်း (The Destruction)Linux VM ကို ပုံမှန်အတိုင်း ဖွင့်ပါ။ Terminal ကို ဖွင့်ပြီး GRUB ရဲ့ Configuration ဖိုင်တွဲကို အမြစ်ပြတ်အောင် ဖျက်ချလိုက်ပါမယ်။ (တကယ့် Hacker တွေ လုပ်သလိုပေါ့)။Bashsudo rm -rf /boot/grub
ဒီ Command ရိုက်ပြီးတာနဲ့ စက်ကို sudo reboot လုပ်ပြီး Restart ချလိုက်ပါ။ဘာဖြစ်သွားမလဲ: စက်ပြန်ပွင့်လာတဲ့အခါ Linux Desktop ကြီး တက်မလာတော့ပါဘူး။ Screen အမည်းကြီးပေါ်မှာ error: no such device သို့မဟုတ် grub rescue> ဆိုတဲ့ စာသားကြီးပဲ ပေါ်ပြီး ရပ်နေပါလိမ့်မယ်။ (ကွန်ပျူတာ သေသွားပါပြီ)။[အဆင့် ၂] ကယ်တင်ခြင်းနှင့် ပြန်ပြင်ခြင်း (The Recovery)ကဲ... အခု သေသွားတဲ့ Linux စက်ကြီးကို IT Support သမားတစ်ယောက်လို ပြန်ရှင်အောင် လုပ်ကြပါစို့။1.Live USB ထည့်ပြီး Boot တက်ခြင်း:VMware ရဲ့ CD/DVD setting ထဲမှာ Linux .iso ဖိုင်ကို ပြန်ချိတ်ပြီး စက်ကို ဖွင့်ပါ။ စက်စဖွင့်ချင်း BIOS (သို့မဟုတ် Boot Menu) ထဲဝင်ပြီး USB/CD ကနေ အရင်ပွင့်အောင် (Boot First) ရွေးပေးပါ။ ဒါဆိုရင် USB ထဲက "Try Ubuntu" (ယာယီ Linux) ထဲကို ရောက်သွားပါမယ်။2.မိမိ၏ SSD Partition ကို ရှာဖွေခြင်း:Live Linux ရဲ့ Terminal ကို ဖွင့်ပြီး မိမိရဲ့ ပျက်နေတဲ့ SSD နာမည်က ဘာလဲဆိုတာကို အောက်ပါ Command နဲ့ ရှာပါ။Bashsudo fdisk -l
အများအားဖြင့် /dev/sda1 သို့မဟုတ် /dev/nvme0n1p1 ဆိုပြီး Linux Filesystem တင်ထားတဲ့ အပိုင်းကို တွေ့ရပါလိမ့်မယ်။3.SSD ကို လှမ်းချိတ်ခြင်း (Mount & chroot):အခု ပျက်နေတဲ့ SSD ထဲကို စုန်းဝင်ပူးဖို့ (chroot လုပ်ဖို့) အတွက် လှမ်းချိတ်ရပါမယ်။Bashsudo mount /dev/sda1 /mnt
sudo chroot /mnt
(ဒီ Command ရိုက်ပြီးတာနဲ့ သင်ဟာ USB ထဲမှာ မဟုတ်တော့ဘဲ ပျက်နေတဲ့ စက်ထဲက SSD ထဲကို တိုက်ရိုက် ရောက်သွားပါပြီ)4.GRUB ကို အသစ်ပြန်မောင်းနှင်ခြင်း:SSD ထဲ ရောက်သွားပြီဖြစ်လို့ ပျက်စီးသွားတဲ့ GRUB တံခါးစောင့်ဖိုင်တွေကို အသစ်ပြန်ဆောက်ခိုင်းရပါမယ်။Bashgrub-install /dev/sda
update-grub
ဒါဆိုရင် စက်ထဲက Kernel တွေကို GRUB က ပြန်ရှာတွေ့သွားပြီး Boot ဆောက်တာ ပြီးဆုံးသွားပါပြီ။5.ထွက်ပြီး Restart ချခြင်း:exit ရိုက်ပြီး chroot ထဲက ထွက်ပါ။ ပြီးရင် စက်ကို Restart ချပြီး Live USB ကို ဖြုတ်လိုက်ပါ။

##### BIOS Settings ရဲ့ အဓိက Concept (၃) ချက်
၁။ The Boot Control Concept (ဘယ်လမ်းကနေ သွားမလဲ)
သဘောတရား: ကွန်ပျူတာ နိုးလာတာနဲ့ Operating System (OS) ရှာဖို့ ဘယ်တံခါးကို အရင်ခေါက်ရမလဲဆိုတာ သတ်မှတ်ပေးတာပါ။

အဓိက ဆက်တင်များ: Boot Order / Boot Priority

IT Support အမြင်: SSD အဟောင်းထဲက OS အတိုင်း တက်မှာလား၊ သို့မဟုတ် ကျွန်တော်တို့ အသစ်စိုက်လိုက်တဲ့ Windows/Linux Live USB ကနေ တက်မှာလားဆိုတာကို ဦးစားပေးရွေးချယ်ခိုင်းတဲ့ Concept ဖြစ်ပါတယ်။

၂။ The Hardware Mode Concept (ပစ္စည်းတွေကို ဘယ်လို ပုံစံနဲ့ သုံးမလဲ)
သဘောတရား: တပ်ထားတဲ့ Hardware ရဲ့ စွမ်းဆောင်ရည်တွေကို အဖွင့်အပိတ် လုပ်တာ သို့မဟုတ် ပုံစံပြောင်းတာ ဖြစ်ပါတယ်။

အဓိက ဆက်တင်များ:

Virtualization (Intel VT-x / AMD-V): CPU ရဲ့ ပါဝါကို ခွဲဝေပြီး VMware ထဲမှာ စက်အသစ်တွေ ထပ်ဆောက်ခွင့် ပေးမလား (Enabled)၊ မပေးဘူးလား (Disabled)။

SATA/Storage Mode (AHCI / RAID): Hard Drive တွေကို ကွန်ပျူတာက ဘယ်လိုနည်းပညာစနစ်နဲ့ ဖတ်မလဲ။

IT Support အမြင်: ဒါကို သိထားရင် "ကျွန်တော့် VMware စက် ဖွင့်မရလို့" ဆိုတဲ့ ဝန်ထမ်းကို BIOS ထဲမှာ Virtualization သွားဖွင့်ပေးရမယ်ဆိုတဲ့ ဖြေရှင်းနည်း ချက်ချင်း ခေါင်းထဲ ပေါ်လာမှာ ဖြစ်ပါတယ်။

၃။ The Security Concept (ဘယ်သူတွေကို ပေးဝင်မလဲ)
သဘောတရား: ကွန်ပျူတာ အောက်ခြေအဆင့် (OS မတက်ခင်) မှာတင် ဘေးကင်းလုံခြုံအောင် ကာကွယ်ထားတဲ့ စနစ် ဖြစ်ပါတယ်။

အဓိက ဆက်တင်များ:

Secure Boot / TPM 2.0: ဗိုင်းရပ်စ်တွေ၊ ဟက်ကာတွေက OS မတက်ခင် အောက်ခြေကနေ ခိုးဝင်တာမျိုး မလုပ်နိုင်အောင် လုံခြုံရေး စစ်ဆေးတဲ့စနစ် (Windows 11 တင်ရင် ဒါတွေ မဖြစ်မနေ လိုပါတယ်။)

BIOS Password: BIOS settings ထဲကို မဆိုင်တဲ့သူ ဝင်မကလိနိုင်အောင် သော့ခတ်ထားတာ ဖြစ်ပါတယ်။


###### BiOS Touch and Dont touch  🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️🛠️
၁။ Really Touch (စိတ်ကြိုက် ကလိလို့ရသော အပိုင်းများ)ဒီအပိုင်းတွေကတော့ အလုပ်အတွက် မဖြစ်မနေ ပြင်ရတတ်ပြီး၊ မှားပြင်မိရင်တောင် စက်မပျက်ဘဲ BIOS ထဲပြန်ဝင်ပြီး အလွယ်တကူ ပြန်ပြင်လို့ရတဲ့အရာတွေ ဖြစ်ပါတယ်။Boot Order / Boot Priority: (USB ကနေ အရင်တက်မလား၊ SSD ကနေ တက်မလား ရွေးတာ)။Intel Virtualization Technology / AMD-V: (VMware မောင်းဖို့ ဖွင့်တာ)။System Date and Time: (စက်ရဲ့ အချိန် ပြင်တာ)။
🔴 ၂။ Really DON'T Touch (လုံးဝ မထိသင့်သော အပိုင်းများ)ဒီအပိုင်းတွေကို မှားပြင်မိရင်တော့ Screen အမည်းကြီးပဲ ပြပြီး BIOS ထဲတောင် ဝင်လို့မရတော့ဘဲ စက်သေသွားတတ်ပါတယ် (Black Screen / No POST ဖြစ်သွားတတ်တယ်)။CPU Frequency / Voltage (Overclocking): CPU ရဲ့ လျှပ်စစ်ဗို့အားနဲ့ အလုပ်လုပ်နှုန်းကို အတင်းမြှင့်တာ။ (မှားရင် CPU လောင်သွားနိုင်သလို၊ စက်လုံးဝ ပွင့်မလာတော့ပါဘူး)။RAM Timings / Memory Voltage: RAM ရဲ့ အလုပ်လုပ်နှုန်းကို အတင်းမြှင့်တာ။ (စက်ဖွင့်တာနဲ့ အဆက်မပြတ် အော်မြည်နေပြီး Screen ပေါ်မှာ ဘာမှ ပြတော့မှာ မဟုတ်ပါဘူး)။BIOS Flashing / Updating (Firmware): BIOS ကို ဗားရှင်းအသစ်တင်နေတုန်း မီးပိတ်လိုက်တာ သို့မဟုတ် ဖိုင်မှားတင်တာ။ (ဒါကို Bricked ဖြစ်တယ်လို့ ခေါ်ပြီး Motherboard ပါ လဲရတတ်ပါတယ်)။
🛠️ ၃။ အဖြစ်အများဆုံး Error ကို တမင်လုပ်ပြီး ပြန်ပြင်ခြင်း (The Storage Mode Error)IT Support နယ်ပယ်မှာ အဖြစ်အများဆုံး BIOS Error တစ်ခုကို တမင်တကာ လုပ်ကြည့်ပြီး (Destroy)၊ ပြန်ပြင်ကြပါမယ် (Solve)။ ဒါကို အပြင်စက်မှာ စမ်းသပ်လို့ ရပါတယ်။ (မီးမလောင်ပါဘူး၊ စက်မပျက်ပါဘူး)။
❌ အဆင့် ၁ - တမင်တကာ Error ဖြစ်အောင်လုပ်ခြင်း (The Destruction)ကွန်ပျူတာကို Restart ချပြီး F2 သို့မဟုတ် Del နှိပ်ကာ BIOS ထဲဝင်ပါ။Advanced သို့မဟုတ် Storage Configuration အောက်က SATA Mode (သို့မဟုတ်) Storage Controller Mode ကို လိုက်ရှာပါ။အခု သင့်စက်မှာ AHCI သို့မဟုတ် NVMe လို့ ဖြစ်နေပါလိမ့်မယ်။ အဲဒါကို တမင်တကာ RAID သို့မဟုတ် IDEMode သို့ ပြောင်းပစ်လိုက်ပါ။F10 နှိပ်ပြီး Save လုပ်ကာ ထွက်လိုက်ပါ။ဘာဖြစ်သွားမလဲ (The Error): စက်ပြန်ပွင့်လာတဲ့အခါ OS Logo (Windows သို့မဟုတ် Linux) ပေါ်လာပြီးတာနဲ့ ချက်ချင်း Blue Screen (BSOD) တက်ပြီး INACCESSIBLE_BOOT_DEVICE ဆိုတဲ့ Error ကြီးနဲ့ စက်က အလိုအလျောက် Restart ပြန်ကျသွားပါလိမ့်မယ်။ ဘယ်လိုမှ OS ထဲ ဝင်လို့ရတော့မှာ မဟုတ်ပါဘူး။အကြောင်းရင်း: Kernel က RAM ပေါ်ရောက်သွားတဲ့အခါ SSD ကို လှမ်းဖတ်ဖို့ ကြည့်တော့ BIOS က Mode ပြောင်းပစ်လိုက်လို့ SSD ကို ရှာမတွေ့တော့ဘဲ Crash ဖြစ်သွားတာပါ။
အဆင့် ၂ - ပြန်လည်ဖြေရှင်းနည်း (The Solution)ဒီပြဿနာကို OS အသစ်ပြန်တင်စရာမလိုဘဲ ၁ မိနစ်အတွင်း ပြန်ပြင်လို့ရပါတယ်။1.BIOS ထဲ ပြန်ဝင်ပါ:စက်ကို ပိတ်ပြီး ပြန်ဖွင့်ပါ။ ဖွင့်ချင်း F2 သို့မဟုတ် Del ကို အဆက်မပြတ်နှိပ်ပြီး BIOS ထဲကို ပြန်ဝင်ပါ။2.မူလ Setting အတိုင်း ပြန်ပြောင်းပါ:Storage Configuration ထဲကို ပြန်သွားပြီး ခုနက တမင်ပြောင်းခဲ့တဲ့ RAID သို့မဟုတ် IDE နေရာမှာ မူလအတိုင်း AHCI (သို့မဟုတ် NVMe) ပြန်ရွေးပေးလိုက်ပါ။3.Save လုပ်ပြီး ထွက်ပါ:ကီးဘုတ်က F10 ကို နှိပ်ပြီး "Save Changes and Exit" ကို ရွေးပါ။ဒါဆိုရင် ကွန်ပျူတာဟာ Blue Screen မတက်တော့ဘဲ ပုံမှန်အတိုင်း Desktop ပေါ်ကို ပြန်ပွင့်လာတာကို တွေ့ရပါလိမ့်မယ်ဗျာ။

###### 💡 ( ဖတ်ရတဲ့အချက်လက်တေသာဖစ်ပီး တကယ်အလုပ်ဖစ်မဖစ် အာမမခံပါ)
အရေးပေါ် ကယ်ဆယ်ရေးနည်းလမ်း (Clear CMOS)တကယ်လို့ သင်က Really DON'T Touch ထဲကအရာတွေကို မှားကလိမိလို့ Screen အမည်းကြီးပဲ ပြနေပြီး BIOS ထဲတောင် ဝင်မရတော့ရင် ဘာလုပ်မလဲ?ဖြေရှင်းနည်း: ကွန်ပျူတာထဲက ပါဝါကြိုးတွေအကုန်ဖြုတ်၊ Motherboard ပေါ်မှာရှိတဲ့ "ကျောက်ကပ်နာရီဓာတ်ခဲ အဝိုင်းလေး" (CMOS Battery) ကို ၅ မိနစ်လောက် ဖြုတ်ထားလိုက်ပါ။၅ မိနစ်ပြည့်ရင် ပြန်တပ်ပြီး စက်ကို ဖွင့်လိုက်ပါ။ BIOS ထဲက Settings တွေအကုန်လုံးဟာ စက်ရုံထုတ် မူလအတိုင်း (Factory Defaults) ပြန်ဖြစ်သွားပြီး Screen ပြန်ပွင့်လာပါလိမ့်မယ်  

##### BIOS Update တင်ရန် တကယ်လိုအပ်သော အခြေအနေများ
၁။ CPU အသစ် လဲချင်တဲ့အခါ (New CPU Support)
ဥပမာ: သင့်မှာရှိတဲ့ Motherboard က အဟောင်း၊ ဒါပေမယ့် စျေးကွက်ထဲ ခုမှအသစ်ထွက်လာတဲ့ CPU အသစ်တစ်လုံးကို ဝယ်ပြီး လာစိုက်ချင်တဲ့အခါမျိုးပါ။ Motherboard က Hardware ပစ္စည်းအသစ်ကို မသိသေးတဲ့အတွက် BIOS Update တင်ပေးမှသာ CPU အသစ်ကို သိပြီး စက်ပွင့်လာမှာ ဖြစ်ပါတယ်။

၂။ ဆိုးရွားတဲ့ Hardware ပြဿနာ (Bug Fixes) ရှိနေတဲ့အခါ
စက်က အကြောင်းရင်းမရှိဘဲ ခဏခဏ Blue Screen တက်နေလို့ ကုမ္ပဏီ (ဥပမာ - Dell, HP) ရဲ့ Website မှာ သွားစစ်တဲ့အခါ "ဒီမော်ဒယ်တွေမှာ ဖြစ်တတ်တဲ့ Error ကို ပြင်ဆင်ရန် BIOS ဗားရှင်းအသစ် တင်ပါ" လို့ တရားဝင် ရေးထားတဲ့အခါမျိုးမှာ တင်ရပါမယ်။

၃။ အရေးကြီးတဲ့ လုံခြုံရေးအပေါက်အပြဲ (Security Vulnerabilities) ရှိလာတဲ့အခါ
တစ်ခါတလေ CPU သို့မဟုတ် Motherboard အောက်ခြေ Chipset တွေမှာ ဟက်ကာတွေ ခိုးဝင်နိုင်တဲ့ အလွန်အန္တရာယ်ကြီးတဲ့ အပေါက်အပြဲတွေ (ဥပမာ - Spectre, Meltdown စတဲ့ ဟာ့ဒ်ဝဲလ်ပိုင်းဆိုင်ရာ လုံခြုံရေးပြဿနာမျိုးတွေ) ပေါ်လာရင် ကုမ္ပဏီက ကာကွယ်ဖို့အတွက် BIOS Update ထုတ်ပေးတတ်ပါတယ်။ အဲဒီအခါမှာ တင်ရပါမယ်။

၄။ RAM အသစ်အမျိုးအစား သို့မဟုတ် NVMe SSD အသစ်တွေနဲ့ တွဲသုံးချင်တဲ့အခါ (Compatibility)
RAM အသစ်စိုက်တာ စက်က မသိတာမျိုး၊ SSD အသစ်အမြန်နှုန်းကို Motherboard က အပြည့်မထုတ်ပေးနိုင်တာမျိုး ကြုံရရင် BIOS အသစ်တင်လိုက်တာနဲ့ အဆင်ပြေသွားတတ်ပါတယ်။

#### ⚠️ BIOS Update တင်ရင် ဘာလို့ အန္တရာယ်များတာလဲ။
Windows Update တင်နေတုန်း မီးပိတ်သွားရင် Windows ပဲပျက်မှာမို့လို့ OS ပြန်တင်ရင် ပြန်ကောင်းပါတယ်။ ဒါပေမယ့် BIOS Update တင်နေတုန်း ၅ မိနစ်အတွင်း မီးပြတ်သွားရင် သို့မဟုတ် စက်ပိတ်သွားရင် ကွန်ပျူတာရဲ့ ဦးနှောက်သေသွားပါပြီ (Bricked ဖြစ်သွားပြီ)။ Motherboard ပေါ်က ချစ်ပ်ပြားကို စက်ရုံသုံးစက်တွေနဲ့ ပြန်ဖျက်ပြီး အသစ်ပြန်ရေးရမှာ ဖြစ်လို့ သာမန်ပြင်ရတာထက် အများကြီး လက်ဝင်သွားပါတယ်။

IT Support အကြံပြုချက်: BIOS Update တင်တော့မယ်ဆိုရင် Laptop မှာဆိုရင် ဘက်ထရီ အပြည့်ရှိနေပါစေ၊ Desktop မှာဆိုရင် မီးမပြတ်အောင် UPS (မီးသိုလှောင်ဗူး) တပ်ထားပြီးမှ တင်တာ အကောင်းဆုံး ဖြစ်ပါတယ်ဗျာ။
