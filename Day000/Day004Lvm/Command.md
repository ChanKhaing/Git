အဆင့်ဆင့် ပြန်ခွဲပေးတာ က နောက် ကျ data တေ ထပ်တိုးချင်တိုင်း ပြန်ပီး boot တေအသစ် ထပ်တက်စရာမလိုတော့တဲ့အတွက်

virtual box ကနေ snapshort လုပ်တာတေ clone တာ သင်ယူခဲံရသလို
snapshort ကို ဒီလို နားလည်ခဲ့ပါတယ်
အချိန် ၄ (Snapshot မရိုက်ခင်)စက်ထဲမှာ /home/chan ဆိုတဲ့ folder ရှိနေတယ်။ (test.txt မရှိသေးဘူး)။အချိန် ၅ (Snapshot ရိုက်တဲ့အချိန်)/home/chan/test.txt ဆိုတဲ့ ဖိုင်ကို ထည့်လိုက်တယ်။ပြီးတော့မှ "Snapshot-5" ကို ရိုက်လိုက်တယ်။💡 ဒီနေရာမှာ VirtualBox ဘာလုပ်လိုက်လဲဆိုတော့: အချိန် ၅ အထိ ရှိခဲ့တဲ့ စက်ရဲ့ အခြေအနေ (Folder တွေ၊ test.txt ဖိုင်တွေ) အားလုံးကို ဓာတ်ပုံရိုက်ပြီး သိမ်းထားလိုက်တာ ဖြစ်ပါတယ်။အချိန် ၆ (လက်ရှိ စမ်းသပ်နေတဲ့အချိန်)သင်က စက်ထဲမှာ ဖျက်ချင်ရာဖျက်၊ လျှောက်လုပ်လိုက်တယ်။ (ဥပမာ- test.txt ကို ဖျက်ပစ်လိုက်တယ်)။❓ အခု မေးခွန်းရဲ့ အဓိက အဖြေ - "Snapshot ကို ဖျက်လိုက်ရင် ဘာဖြစ်မလဲ?"ဒီနေရာမှာ "Snapshot ကို ဖျက်တာ (Delete)" နဲ့ "Snapshot ကို ပြန်သွားတာ (Restore/Rollback)" ရဲ့ ကွာခြားချက်ကို သိထားဖို့ လိုပါတယ်။လုပ်ဆောင်ချက်ရလဒ်Restore လုပ်ရင် (ပြန်သွားရင်):အချိန် ၆ မှာ ဘာတွေပဲ ဖျက်ခဲ့ဖျက်ခဲ့... Snapshot-5 ကို Restore လုပ်လိုက်ရင် အချိန် ၅ တုန်းက test.txt ရှိခဲ့တဲ့ အခြေအနေဆီကို ပြန်ရောက်သွားပါလိမ့်မယ်။ (ဒါက သင်သိထားတဲ့အတိုင်း မှန်ပါတယ်)။Delete လုပ်ရင် (ဖျက်လိုက်ရင်):"ဒီ Snapshot-5 ဆိုတဲ့ မှတ်တမ်းကို ငါမလိုချင်တော့ဘူး၊ ဖျက်လိုက်တော့" လို့ ပြောတာဖြစ်လို့ Snapshot-5 ဆိုတဲ့ မှတ်တမ်းဖိုင်ပဲ ပျောက်သွားမှာပါ။ သင့်စက်ရဲ့ အခြေအနေကတော့ အချိန် ၄ ကို ပြန်မသွားဘဲ လက်ရှိ အချိန် ၆ (သင် ဖျက်ချင်ရာဖျက်ထားတဲ့ ပျက်စီးနေတဲ့ အခြေအနေ) အတိုင်းပဲ ဆက်ရှိနေမှာ ဖြစ်ပါတယ်။

VirtualBox မှာ Snapshot တစ်ခု ရိုက်လိုက်ပြီဆိုတာနဲ့ သူက မူရင်း Disk ဖိုင်ကြီးကို ခေတ္တ ပိတ်လိုက်ပြီး "စာအုပ်အသစ် (Difference Disk)" တစ်အုပ် ထပ်တွဲပေးလိုက်တာ ဖြစ်ပါတယ်။

သင် အချိန် ၆ မှာ လုပ်သမျှ အပြောင်းအလဲတွေ (ဖျက်တာ၊ တိုးတာ) အားလုံးကို အဲဒီ စာအုပ်အသစ်ထဲမှာပဲ ရေးပါတယ်။

သင်က Snapshot ကို Delete (ဖျက်) လိုက်ပြီ ဆိုရင် VirtualBox က အဲဒီ စာအုပ်အသစ်ထဲက ဒေတာတွေကို မူရင်း disk ထဲကို သွားပြီး ပေါင်းထည့် (Merge) ပစ်လိုက်ပါတယ်။ ဒါကြောင့်မို့လို့ လက်ရှိ အခြေအနေအတိုင်းပဲ ဆက်ရှိနေတာ ဖြစ်ပြီး အချိန် ၄ ဆီကို နောက်ပြန်ဆုတ်သွားခြင်း မရှိတာ ဖြစ်ပါတယ်။

အတိုချုပ်မှတ်ရန်: အချိန်ဟောင်းကို ပြန်သွားချင်ရင် "Restore" နှိပ်ရမှာ ဖြစ်ပြီး၊ သိမ်းထားတဲ့ မှတ်တမ်းမှတ်ရာကို မလိုချင်တော့လို့ ဖျက်ပစ်ချင်မှသာ "Delete" နှိပ်ရမှာ ဖြစ်ပါတယ်

setting ကနေ hard disk physical volume တိုးတာတေ attachemet မဖစ်တဲ့ hard disk ပြန်ဖျက်တာကိုလည်း သင်ရပါတယ်
သင်ယူမှုအတိုင်းမဖစ်တဲ့ ဟာတေကိုလည်း minimal xubuntu မာ တွေ့ရပါတယ် 
swap ပဲရှိပီး ကျန်တဲ့ အသစ်တေ ဖစ်တဲံ Ext4 XFS Btrsf Vfat တေကို တွေ့ခဲ့ ရပါတယ် ဒါတေကို ဖတ်ဖစ်ပါတယ်
။ Ext4 (Fourth Extended Filesystem)
အဓိပ္ပာယ်: Linux လောကမှာ အသုံးအများဆုံးနဲ့ အငြိမ်ဆုံး (Stable) စံသတ်မှတ်ချက် ဖိုင်စနစ် ဖြစ်ပါတယ်။

ဘယ်နေရာမှာ သုံးမလဲ: Linux ရဲ့ Main Root (/) နဲ့ /boot အပိုင်းတွေအတွက် အကောင်းဆုံး ဖြစ်ပါတယ်။ သင့်အနေနဲ့ ဘာရွေးရမှန်း ဝေခွဲမရရင် Ext4 ကိုပဲ ရွေးချယ်ဖို့ အကြံပြုပါတယ်။

၂။ XFS
အဓိပ္ပာယ်: Data ပမာဏ အကြီးကြီးတွေနဲ့ ဖိုင်ဆိုဒ် အကြီးကြီးတွေကို ကိုင်တွယ်တဲ့နေရာမှာ အထူးကောင်းမွန်တဲ့ high-performance ဖိုင်စနစ် ဖြစ်ပါတယ်။ Enterprise ဆာဗာကြီးတွေ (ဥပမာ- Red Hat Linux) မှာ အသုံးများပါတယ်။

ဘယ်နေရာမှာ သုံးမလဲ: သင့်ကွန်ပျူတာမှာ Storage ပမာဏ အများကြီးရှိပြီး (Terabytes ချီပြီး) ဖိုင်ကြီးတွေကိုပဲ အဓိက သိမ်းဆည်းမှာဆိုရင် သုံးနိုင်ပါတယ်။ သာမန် Desktop သုံးအတွက်တော့ သိပ်မလိုပါဘူး။

۳။ Btrfs (Better FS / Butter FS)
အဓိပ္ပာယ်: ခေတ်မီ Features တွေပါတဲ့ ဖိုင်စနစ်သစ် ဖြစ်ပါတယ်။ သူက System တစ်ခုလုံးကို backup လုပ်ရလွယ်ကူစေတဲ့ Snapshot ရိုက်တာမျိုး၊ Partition ဆိုဒ်တွေကို လိုသလို အလွယ်တကူ တိုး/လျှော့ လုပ်တာမျိုးတွေကို Installer ပြင်ပကနေ အလွယ်တကူ လုပ်နိုင်ပါတယ်။

ဘယ်နေရာမှာ သုံးမလဲ: Advance သုံးချင်သူတွေ၊ System တစ်ခုခုဖြစ်ရင် Snapshots ကနေ အမြန် ပြန် recovery လုပ်ချင်သူတွေ ရွေးချယ်ကြပါတယ်။ (Xubuntu မှာတော့ Ext4 က ပိုအဆင်ပြေပါတယ်)။

၄။ VFAT (FAT32)
အဓိပ္ပာယ်: Windows ကော Linux မှာပါ အလုပ်လုပ်တဲ့ အရင်ခေတ်က File System အဟောင်းတစ်ခု ဖြစ်ပါတယ်။

ဘယ်နေရာမှာ သုံးမလဲ: အခုခေတ် ကွန်ပျူတာအသစ်တွေရဲ့ UEFI Boot စနစ်အတွက် လိုအပ်တဲ့ EFI System Partition (ESP) ဆောက်တဲ့နေရာမှာပဲ ဒီကောင်ကို မဖြစ်မနေ ရွေးပေးရလေ့ရှိပါတယ်။

၅။ Swap
အဓိပ္ပာယ်: ဒါကတော့ File System မဟုတ်ဘဲ Virtual RAM (RAM အတု) အဖြစ် သုံးဖို့အတွက် ဖြစ်ပါတယ်။ ကွန်ပျူတာမှာ RAM မလောက်တော့တဲ့အခါ Hard disk ပေါ်က ဒီ partition နေရာကို RAM အဖြစ် ယူသုံးပါတယ်။

၆။ Leave unformatted
အဓိပ္ပာယ်: "ဘာ format မှ မချဘဲ ဒီအတိုင်း ထားလိုက်ပါ" လို့ ပြောတာ ဖြစ်ပါတယ်။

ဘယ်နေရာမှာ သုံးမလဲ: သင်က LVM (Volume Group) ဆောက်ဖို့အတွက် space သက်သက်ပဲ ဖယ်ထားချင်တာမျိုး၊ ဒါမှမဟုတ် တခြား OS တစ်ခုခုက သုံးနေတဲ့ partition မို့လို့ Data တွေ မပျက်စေချင်တဲ့အခါမျိုးမှာ ဒီ option ကို ရွေးရပါတယ်။
